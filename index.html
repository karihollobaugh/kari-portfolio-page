Full-Stack Running Songs Database App

Progress Update!

I’m excited to share that my running songs database app is officially up and running! This app is designed for runners who want to curate and organize high-energy playlists based on BPM, genre, and artist. Whether you're looking for the perfect track to match your pace or want to discover new songs that keep you moving, this app makes it easy to store and retrieve all the best jams. Here's a quick rundown:

  • Tech Stack: Java (Spring Boot) + PostgreSQL
  • Backend: I’ve got a RESTful API set up to handle GET and POST requests for managing songs in my DB.
  • Database: PostgreSQL is connected, and I'm storing song details (like title, artist, genre, and BPM) with auto-generated IDs.
  • Success: After battling through some frustrating 500 errors and debugging endpoint/data type issues, I finally got that glorious 200 OK status! ✅

Next up: creating the front end... and dare I say, integrating an API?! Stay tuned!
